Design tokens

Token set used by this static clone

The values below intentionally echo the analyzed Bento-detail page: clean white surfaces, blue-forward hierarchy, `Space Grotesk`, and a sharp no-radius silhouette.

Color system

Background `hsl(0 0% 100%)` / pure white canvas
Foreground `hsl(222 47% 11%)` / deep navy-black text
Primary `hsl(225 100% 50%)` / electric blue accent
Border `hsl(220 13% 91%)` / quiet framing and chip outlines

Typography

Space Grotesk
Display + body family / bold geometric tone / strong for product storytelling
Eyebrow

Uppercase, 0.28–0.38em tracking, medium weight, blue accent.

Hero title

Clamp-driven display scale, tight leading, negative tracking, bold weight.

Body copy

Muted gray foreground with generous line-height to keep dense content breathable.

Layout primitives

  • Radius `0px` to keep the silhouette crisp and system-like.
  • Grid background Two 1px linear gradients form the subtle square lattice.
  • Container `min(1180px, calc(100vw - 48px))` for wide editorial spacing.
  • Shadow Very soft depth only on cards and CTA, never theatrical.
Component spec

Reusable components in this concept page

Each block below documents the role, structure, and styling intent of the clone so the page doubles as its own mini component spec sheet. Sneaky, but respectable.

Hero display

Purpose

Establish product positioning with a typography-first composition and a single decisive CTA.

Structure

Eyebrow → split display title → framed primary phrase → supporting paragraph → action row.

Key treatment

The boxed-corner phrase uses a full border plus four detached corner squares to echo the reference screenshots.

Label system

Purpose

Provide category and state metadata without competing with the main title.

Structure

Inline-flex pill, uppercase text, wide letter spacing, quiet outline. Featured state adds blue tint and icon.

Key treatment

Sharp edges, restrained color, and enough whitespace to feel catalog-like rather than app-like.

Bento tile

Purpose

Tell a dense product story through modules of varied size while preserving a reading order.

Structure

Kicker → short title → supporting copy → metric or keyword footer.

Key treatment

Mixed widths, low-contrast borders, and hover lift only for polish, not for gamification.

Token card

Purpose

Expose the palette, typography, and layout primitives directly in the page for handoff clarity.

Structure

Small uppercase heading plus grouped examples or list items.

Key treatment

Uses the same shell as content cards so documentation feels native to the design, not bolted on.

Primary CTA

Purpose

Drive one clear action from the hero without creating choice overload.

Structure

Single-line label, bold text, filled blue background, large horizontal padding.

Key treatment

No radius, no loud glow, no extra iconography — just confident contrast.

Keyword tags

Purpose

Expose searchable concepts and product themes in a secondary metadata layer.

Structure

Small outlined units with muted text and consistent vertical rhythm.

Key treatment

They support scanning and taxonomy, but never steal attention from core content.
