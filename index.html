Architecture decisions

Engineering foundations, documented as ADRs

Every major design choice is captured in an Architecture Decision Record — immutable rationale that survives team rotation, scope creep, and late-night debugging.

ADR-001

Stack-Based State Machine Execution Engine

A push/pop execution stack with a while-loop scheduler. Supports branches, loops, merges, and sub-workflow invocations — no recursion, no fan-out races. Each node executes exactly once per cycle.

Accepted
ADR-002

Node Output Format — 2D Array

Every node emits outputs[][] — first dimension for output ports, second for items. Generic routing with json, meta, error, and branch fields enables a uniform data contract across all node types.

Accepted
ADR-003

Loop Node — SplitInBatches

Stateful iterator pattern with nodeContext storage. Items are consumed in configurable batches, and re-queuing cycles allow downstream sub-graphs to process each batch before the loop advances.

Accepted
ADR-004

Editor State Architecture

Centralized workflowEditor service owns all canvas state. Mutations go through actions.* only. Undo/redo with command batching, multi-tab isolation, and service-injection across all OWL components.

Proposed
ADR-005

Zero Trust + Principle of Least Privilege

Row-level security, run_as_user execution context, SafeEnvProxy sandboxing, and secret brokering — workflows execute only with the permissions their owner possesses, never as superuser.

Accepted
ADR-006

Version History — Patch Storage

50-version FIFO with line-level patches. Milestone protection prevents accidental overwrites. Full reconstruction from any saved patch chain, with metadata for diff display.

Accepted
ADR-008

Hybrid Trigger Architecture

Canvas trigger nodes bridge to Odoo-native subsystems: ir.cron for schedules, base.automation for record events, UUID-routed controllers for webhooks. All paths converge to _execute_from_trigger() — one entry point, many activation surfaces.

Accepted
Security model

Zero Trust from day one, not bolted on later

Automation platforms are high-value targets because they run with accumulated permissions. Workflow Studio inverts that pattern — every execution context is scoped, sandboxed, and auditable.

01

Row-Level Security

ORM access rules enforced at every read/write — no superuser bypass in workflow context.

02

run_as_user

Workflows execute with the original user's permissions, even when triggered asynchronously.

03

SafeEnvProxy

Expression evaluation runs inside a restricted proxy — no model browsing, no shell escape.

04

Secret Brokering

API keys and tokens are resolved at execution time through a vault layer, never stored in workflow JSON.

05

Audit Trail

Every execution produces a run record with step-level timing, user context, and error snapshots.

Roadmap

From MVP to production, in milestones

An honest roadmap — not a marketing timeline. Each phase ships a usable increment with clear acceptance criteria.

Phase 1 — MVP

Core Engine + Editor

A working visual editor and execution engine inside Odoo, with manual and scheduled triggers, core node types, and expression evaluation.

  • Stack-based execution engine
  • Visual node editor (OWL canvas)
  • Manual + Schedule + Webhook triggers
  • HTTP Request, IF/Switch, Loop nodes
  • Expression system ({{ $json.field }})
  • Execution run history & step inspector
Phase 2 — Beta

Production Hardening

Error handling, version history, permission model, and queue-based async execution for long-running workflows.

  • Patch-based version control (ADR-006)
  • Zero Trust security model (ADR-005)
  • Queue-job async execution
  • Error boundaries & retry policies
  • Record event triggers (base.automation)
  • Observability & debugging tools
Phase 3 — Production

Scale & Ecosystem

Multi-tenant deployment, community node marketplace, workflow templates, and enterprise governance features.

  • Sub-workflow invocations
  • Community node registry
  • Workflow templates & cloning
  • Rate limiting & resource quotas
  • Multi-company isolation
  • Enterprise audit & compliance
